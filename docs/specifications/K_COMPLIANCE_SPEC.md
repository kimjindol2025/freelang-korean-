# K-Compliance v1.0 - 한국형 규정 준수 표준 라이브러리

**상태**: Draft Specification v0.1
**작성일**: 2026-03-26
**목표**: 대한민국 법령(PIPA, ISMS, PCI-DSS) 준수 표준화

---

## 📋 개요

K-Compliance는 **한국 정보보호 관련 법령을 코드 수준에서 표준화**하는 K-FreeLang의 핵심 모듈입니다.

### 규범 근거
- **PIPA** (개인정보보호법): 개인정보 취급 기준
- **ISMS** (정보보안관리체계): 보안 인증 체계
- **PCI-DSS** (결제카드산업 데이터 보안 기준): 금융 거래 보안

## 🎯 핵심 함수 목록 (표준 명칭 확정)

### 1️⃣ 신원 검증 (신원_검증 모듈)

#### 1-1. 주민등록번호 검증
```freelang
함수 주민등록번호_유효성확인(rrn: 문자열) -> 참거짓
```
**용도**: 대한민국 주민등록번호의 체크디지트 검증
**입력**: "123456-1234567" 형식
**반환**: 유효성 여부 (참/거짓)
**규범**: 통계청 고시 '주민등록번호 부여·관리 규칙'

#### 1-2. 사업자등록번호 검증
```freelang
함수 사업자등록번호_유효성확인(brn: 문자열) -> 참거짓
```
**용도**: 대한민국 사업자등록번호 검증
**입력**: "123-45-67890" 형식
**반환**: 유효성 여부
**규범**: 국세청 '사업자 등록관리 규칙'

#### 1-3. 법인등록번호 검증
```freelang
함수 법인등록번호_유효성확인(crn: 문자열) -> 참거짓
```
**용도**: 대한민국 법인의 고유식별번호 검증

#### 1-4. 휴대폰번호 검증
```freelang
함수 국내휴대폰번호_유효성확인(phone: 문자열) -> 참거짓
```
**용도**: 010/011/016/017/018/019 형식 검증
**입력**: "010-1234-5678" 또는 "01012345678"

#### 1-5. 전화번호 검증
```freelang
함수 국내전화번호_유효성확인(phone: 문자열) -> 참거짓
```
**용도**: 02/031 등 지역번호 포함 검증

### 2️⃣ 개인정보 보호 (개인정보_보호 모듈)

#### 2-1. 개인정보 비식별화
```freelang
함수 개인정보_비식별화(
  데이터: 문자열,
  유형: 문자열  // "주민번호" | "휴대폰" | "이메일" | "카드번호"
) -> 문자열
```
**용도**: PIPA 제27조 '개인정보의 비식별화 조치'
**예시**:
- "123456-1234567" → "123456-****567"
- "010-1234-5678" → "010-****-5678"
- "user@example.com" → "u***@example.com"
- "1234-5678-9012-3456" → "****-****-****-3456"

#### 2-2. 민감정보 마스킹
```freelang
함수 민감정보_마스킹(
  데이터: 문자열,
  마스킹문자: 문자열  // 기본값: "*"
) -> 문자열
```
**용도**: 민감정보(주민번호, 건강정보, 신용정보) 처리
**규범**: PIPA 제23조 '개인정보의 안전성 확보'

#### 2-3. 개인정보 암호화
```freelang
함수 개인정보_암호화(
  평문: 문자열,
  알고리즘: 문자열  // "ARIA-256" | "SEED-128"
) -> 문자열
```
**용도**: PIPA 준수 암호화 표준
**규범**: 행정안전부 '암호 이용 촉진법'

#### 2-4. 개인정보 복호화
```freelang
함수 개인정보_복호화(
  암호문: 문자열,
  키: 문자열,
  알고리즘: 문자열
) -> 문자열
```

### 3️⃣ 감사 및 로깅 (감사_로그 모듈)

#### 3-1. 접근 로그 기록
```freelang
함수 접근로그_기록(
  사용자ID: 문자열,
  자원: 문자열,
  접근유형: 문자열,  // "읽기" | "쓰기" | "삭제"
  결과: 참거짓
) -> 공집합
```
**용도**: ISMS 'A.12.4.1 사용자 활동 로깅'
**기록 항목**: 시각, 사용자, 자원, 결과, 상태

#### 3-2. 보안 이벤트 로깅
```freelang
함수 보안이벤트_로그(
  이벤트유형: 문자열,  // "인증실패" | "접근거부" | "이상탐지"
  심각도: 문자열,      // "낮음" | "중간" | "높음" | "심각"
  설명: 문자열
) -> 공집합
```
**용도**: ISMS 'A.12.4.1 보안 이벤트 감시'

#### 3-3. 감사 로그 조회
```freelang
함수 감사로그_조회(
  시작일: 문자열,
  종료일: 문자열,
  필터: 문자열  // JSON 형식
) -> 배열<맵<문자열, 문자열>>
```

### 4️⃣ 접근 제어 (접근_제어 모듈)

#### 4-1. 권한 검증
```freelang
함수 권한_확인(
  사용자ID: 문자열,
  권한: 문자열  // "읽기" | "쓰기" | "삭제" | "관리"
) -> 참거짓
```
**용도**: ISMS 'A.6.2 사용자 접근 관리'
**규범**: 최소 권한 원칙 (Principle of Least Privilege)

#### 4-2. 역할 기반 접근 제어 (RBAC)
```freelang
함수 역할_권한_확인(
  사용자ID: 문자열,
  역할: 문자열,  // "관리자" | "사용자" | "게스트"
  자원: 문자열
) -> 참거짓
```

#### 4-3. 세션 타임아웃 검증
```freelang
함수 세션_유효성확인(
  세션ID: 문자열,
  최대유지시간_분: 숫자  // 기본값: 30
) -> 참거짓
```

### 5️⃣ 암호화 및 해싱 (암호화_보안 모듈)

#### 5-1. SHA-256 해싱
```freelang
함수 SHA256_해시화(텍스트: 문자열) -> 문자열
```
**용도**: 비밀번호, 파일 무결성 검증
**규범**: ISMS 'A.10.1.1 암호화 정책'

#### 5-2. ARIA 암호화 (대한민국 표준)
```freelang
함수 ARIA_암호화(
  평문: 문자열,
  키: 문자열
) -> 문자열
```
**규범**: TTA PG6.0 'ARIA 암호 알고리즘'
**국가표준**: KS X 1213 (기술표준원)

#### 5-3. SEED 암호화 (대한민국 표준)
```freelang
함수 SEED_암호화(
  평문: 문자열,
  키: 문자열
) -> 문자열
```
**규범**: TTA PG6.0 'SEED 암호 알고리즘'
**국가표준**: KS X 1214 (기술표준원)

```freelang
함수 PIPA_개인정보수집동의_검증(
```

## 📐 명칭 설계 원칙

### 1. 국립국어원 표준어 준수
- ✅ "주민등록번호" (O) vs "주민번호" (X)
- ✅ "비식별화" (O) vs "마스킹" (영어 차용, 최소화)
- ✅ "유효성확인" (O) vs "벨리데이션" (X)

### 2. 동사형 + 목적어 구조
```
함수 [목적어]_[동사형](파라미터) -> 반환타입
```
- 예: `주민등록번호_유효성확인`, `개인정보_비식별화`, `SHA256_해시화`

### 3. 언더스코어(_) 사용
- 다중 단어 연결: `개인정보_암호화`
- 한글 가독성 최우선

### 4. 한국식 문화 반영
- ARIA, SEED 같은 **한국 국가표준** 암호화 우선 배치
- 주민번호, 사업자번호 같은 **한국 고유 신원 체계** 포함
